Opdracht 8

Styling wordt toegepast in een bepaalde volgorde. Het kan voorkomen dat je styling toepast en vervolgens niet het gewenste resultaat ziet. Cascade en Inheritance zijn belangrijke concepten om te begrijpen hoe styling werkt. Leg in eigen woorden uit wat met cascade en inheritance wordt bedoeld. Maak voorbeelden om je antwoord uit te leggen.
  • Cascade: zit in de naam van CSS: Cascading Style Sheets. Het is van belang wanneer er "conflicting rules" (tegenstrijdige regels) aanwezig zijn in je styling, oftewel een CSS conflict. Welke styling krijgt de voorkeur als er 2 styling regels op hetzelfde element van toepassing zijn? De cascade en het begrip specificiteit bepalen samen welke styling de overhand krijgt. Cascade houdt in dat wanneer twee regels een conflict hebben en zij dezelfde specificiteit hebben (bijv allebei een type selector zijn), dan zal de regel die later in de stylesheet staat bepalend zijn. De volgorde is dus ook: externe styling, interne styling, inline styling, waarbij dus de inline styling later in het document staat dan de andere twee en dus de andere "overwrite". Praktijkvoorbeeld

  • Inheritance: sommige CSS rules zijn zowel van toepassing op het parent element en de child elements, en andere targeten alleen het parent element. Praktijkvoorbeeld

    • Voorbeelden van CSS properties die inheritance vertonen:
      border-collapse, border-spacing, caption-side, color, cursor, direction, empty-cells, font (-size, -family, etc), letter-spacing, line-height, list-style, orphans, quotes, tab-size, text(-size, -align, etc), visibility, white-space, widows, word-break, word-spacing, word-wrap.

    • Voorbeelden van CSS properties zonder inheritance:
      background (-color, -image, etc), border (-color, -style, -widht, etc), clear, content, display, float, height, left, right, top, bottom, margin, outline, padding, width, z-index.
