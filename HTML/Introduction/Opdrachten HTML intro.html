Internet is het fysieke netwerk, world wide web is de verzameling webpagina's die je via het netwerk deelt.

HTML is als het skelet, de structuur van de website, CSS geeft de styling aan, en JavaScript is de taal voor complexe dingen op een website (graphics, interactive maps, etc).

7. Wat is het verschil tussen inline en block elementen?
Block elementen vormen een zichtbaar blok op de website, op een nieuwe regel. Het zijn structuurelementen op de pagina: headings, paragrafen, lijsten, navigatiemenu's, footers.
Inline elementen zitten in block elementen en omgeven alleen kleine stukjes content. Een inline element begint niet op een nieuwe regel, en zijn meestal van toepassing op tekst: anchor element, emphasis, strong, etc.

Semantics

9. Sinds de komst van HTML5 zijn er elementen die betekenissen hebben. Wat wordt er bedoeld met HTML semantics? Waarom is het gebruik van semantics belangrijk?
Sommige HTML tags geven een bepaalde indicatie van het soort content dat ze bevatten, oftewel ze hebben een extra betekenis. Een <h1> tag geeft bijvoorbeeld aan dat de content van dat element de belangrijkste heading van die pagina of sectie is. Dat noemt men HTML semantics. Door middel van correct gebruik van HTML semantics geven we een apparaat (browser, smartphone, etc) de juiste informatie mee.

● Semantic markup requires that HTML elements be used according to their intended purpose.
● Semantic markup requires the separation of content and presentation.
When writing semantic markup, the presentation of web page elements is kept completely separate and distinct from the markup of the content itself. Semantic markup is the use of a markup language such as HTML to convey information about the meaning of each element in a document through proper selection of markup elements, and to maintain complete separation between the markup and the visual presentation of the elements contained in the document. non-semantic elements don't have any information for the computer, such as div and span. id and class attributes are often used to hint at the meaning of the content of non-semantic elements.

Document Structure: semantic elements to be used instead of the div tag:

10. Wat voor content zou je in de volgende elementen gebruiken?
  • <header> used for a web page header which typically contains the site logo, heading elements, and site navigation.
  • <footer> used for a web page footer which typically contains authorship, contact, and copyright information in addition to navigational links and a link back to the top of the web page.
  • <main> A high-level element used to contain all of the content that is unique to a single web page and not repeated across multiple web pages.
  • <nav> An element to contain blocks of site navigation links. This element is typically placed in the page header and footer, and may also be used in an aside (sidebar) element as well.
  • <section> The section element is used to mark off sections of a document, such as chapters or major sections of a long form post.
  • <aside> Use to identify content that is related to the main content on the page but not part of the primary flow of the document. For example, the aside element may contain a glossary definition of a term that appears in a blog post or it may contain advertisements related to the contents of the page.
  • <article> The article element is used to identify a block of content suitable for reuse and syndication in other settings, such as a blog post or technical article.

Textual Meaning: semantic elements to be used instead of the span tag:

  • <em> tags show the computer that this word or sentence has emphasis and it appears in italics.
  • <h1> through <h6>: heading element tags are used to identify text that should appear as a heading, with h1 being the most important one, h2 less important, etc.
  • <strong> adds boldness to the text and is given extra importance.
  • <mark> is used to highlight text of specific importance. For example, to highlight the occurence of a search word on a web page.
  • <cite> is used to identify the original work from which a bit of content originates.
  • <blockquote> or <q> elements are used to identify text that is a direct quotation from another source.
  • <time>element can be used to tell browsers, web crawlers, and other smart devices that a specific bit of content represents time on a 24-hour clock or a specific calendar date.

Media Type: HTML5 has three tags that show what media type is involved:

  • <audio> Used to add one or more sources of audio content to a document and to allow the browser to pick the best option based on the visitor’s device and browser.
  • <video> Similar to the audio element but used to add video content to a markup document.
  • <picture> used to allow a web browser to pick the best image from the available options based on the results of a media query.

Correlation tags: signal a correlation between multiple elements:

  • <u1> Unordered lists are used to signal a relationship between the items on the list and to indicate that they do not need to be understood in a specific order.
  • <figure> used to group together a piece of content, such as an image, chart, graph, or text, and a caption marked off by figcaption tags. By nesting the caption and the content between figure tags a relationship between the nested elements is identified.
  • <address> This attribute is used to associate contact information with the parent element that contains the address element. For example, when added to an article, the address element provides contact information for the article author, and when added to a web page footer the address identifies contact information for the web page owner.
HTML has grown increasingly complex over the last several years, and it can be tempting to keep using div elements with class and id attributes, but the accessibility and interoperability promise of semantic HTML5 tags is reason enough to embrace these new semantic elements.

Headers

Het gebruik van headers is belangrijk voor zoekmachines en de gebruikers van je website.

11. Onderzoek en leg uit waarom headers zowel belangrijk zijn voor zoekmachines als voor de gebruikers.
In de metadata in de HTML head sectie kun je een beschrijving toevoegen die wordt weergegeven als je pagina in een search engine als zoekresultaat naar voren komt. Dat schrijf je als <meta name="description" content="beschrijving van je pagina"> Maar hier gaat de vraag over headers. De header tag loopt van 1 tot 6, waarbij 1 de grootste en tegelijk ook de belangrijkste header is. h1 is de main heading, h2 de sub-heading, h3 de sub-sub-heading, etc. Gebruik bij voorkeur maar 1 h1 header per pagina, gebruik de headers in de juiste volgorde, en gebruik op 1 pagina liever niet meer dan 3 verschillende headers.
Headers zijn van belang voor de gebruiker omdat die in eerste instantie vaak een website alleen scant met zijn ogen en dus vooral op de headers let. Als die ontbreken schrikt dat de gebruiker af. Daarnaast bestaat er voor mensen met beperkt zicht de functie "screen reader", die de gebruiker helpt om de site te navigeren dmv het voorlezen van de headings.
Zoekmachines beschouwen de content van headers als belangrijke keywords om de search rankings van de site te bepalen. Zonder headings doet je pagina het slecht mbt SEO (Search Engine Optimization).

Voor het aanmaken van een text-header, in het Nederlands een subkop of kop, kan je gebruik maken van een Header tag. Je kunt verschillende grootte koppen creëren.

Paragrafen

13. Onderzoek wat een paragraaf tag is en hoe hij gebruikt wordt. Geef hier een voorbeeld bij.
Een gestructureerde tekst bestaat uit kopjes/headings en paragrafen. De headings zijn hiervoor besproken. De paragraaf tag is <p>. Meestal staat onder elke heading een paragraaf. Dmv de headings creëer je een hierarchische structuur. Paragrafen dragen bij aan de structuur. Voorbeeld van een paragraaf: <p>I am a paragraph, oh yes I am.</p> Resultaat:

I am a paragraph, oh yes I am.

14. Onderzoek en leg het verschil tussen het stoppen van alle tekst in één paragraaf tag, en het opsplitsen van de tekst in meerdere tags uit.
Wanneer je alle tekst in 1 paragraaf tag plaatst, wordt het een doorlopende tekst zonder breaks, en dus onoverzichtelijk. Door de stukjes tekst in meerdere paragrafen te verdelen wordt het geheel overzichtelijker, er komen ruimtes tussen de blokjes tekst. Bovendien kun je dan de verschillende stukjes tekst niet verschillend opmaken, omdat het allemaal tot een en dezelfde paragraaf behoort. Dus voor CSS en JavaScript is het ook van belang om de tekst in paragrafen te verdelen.
